-- These define the actual mount of a cavalry unit; i.e. the stats of the mount itself when on its own.
--
-- Every mount form here should have an explicit #command "#gcost X" and #command "#rcost" set. These
-- set the price of the mount form itself, and Dominions adds it to the price of the overall mounted
-- unit when recruiting it.
--
-- Since #copystats can be used on both recruitable and non-recruitable creatures (for example, salamanders
-- which cost 50g in vanilla and scorpion beasts which are a summon and therefore cost 0g), not setting an
-- explicit #gcost makes it harder to parse what the cost of the unit using the mount will be.
--
-- Using #command "#rcost X" will set the resource cost of this mount form. However, Dominions will factor
-- in resource costs of given armors, so there's no need to make a horse and a horse with #command "#armor 252"
-- a different #rcost.
--
-- The #id of the mount form is what is used to "hook up" the mount to the rider. Poses will have a
-- #load statement for a file containing mounts, where they will be defined like an equipped item, in the same
-- way as armours or weapons. These mount item definitions are where the #id of the mount form is referenced,
-- through the #command "#mountmnr '<mount_form_id>'" command.
